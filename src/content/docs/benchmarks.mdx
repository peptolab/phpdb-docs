                <div>
                    <h2 class="type__heading type__heading--sm">Notes</h2>
                    <p class="type">These benchmarks are intended to provide a general comparison. Real-world performance will vary based on your specific use case, database configuration, and query complexity. All libraries were tested with their default configurations and recommended best practices.</p>
                    <p class="type">The benchmark source code is available in the <a href="https://github.com/php-db/phpdb-benchmarks" target="_blank" rel="noopener noreferrer">phpdb-benchmarks repository</a>.</p>
                </div>
                <div>
                    <h2 class="type__heading type__heading--sm">Sakila Database</h2>
                    <p class="type">These benchmarks use the <a href="https://dev.mysql.com/doc/sakila/en/" target="_blank" rel="noopener noreferrer">Sakila sample database</a>, a well-established MySQL reference schema originally developed by Mike Hillyer. Sakila models a DVD rental store with realistic tables including films, actors, customers, rentals, and payments.</p>
                    <p class="type">Sakila is an ideal benchmarking dataset because it represents real-world relational patterns: one-to-many relationships (films to inventory), many-to-many relationships (films to actors via film_actor), and complex joins across multiple tables. With over 1,000 films, 200 actors, and 16,000 rental records, it provides meaningful data volumes while remaining manageable for repeatable testing.</p>
