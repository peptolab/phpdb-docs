## Cross-Database Adapter Layer

A unified interface for MySQL, PostgreSQL, SQLite, and SQL Server with consistent quoting and parameter handling.

phpdb provides a clean abstraction over PDO that handles the subtle differences between database platforms. Whether you're working with MySQL's backtick quoting, PostgreSQL's double quotes, or SQL Server's square brackets, the adapter layer handles it transparently.

Connection management is straightforward with support for lazy connections, connection pooling awareness, and automatic reconnection. The adapter exposes the underlying PDO instance when you need direct access, while providing convenience methods for common operations.

## SQL Query Builders

Structured Select, Insert, Update, Delete objects with full support for typed arguments, nesting, and mutability.

Build complex queries programmatically without string concatenation. The query builders support joins, subqueries, unions, common table expressions, and window functions. Each builder is mutable by default, allowing you to progressively construct queries based on runtime conditions.

Expressions and predicates are first-class objects. Create reusable WHERE clause fragments, combine them with AND/OR logic, and nest them arbitrarily deep. The builders automatically handle identifier quoting and parameter placeholder generation.

## TableGateway Pattern

A simple, structured approach to CRUD and table-level operations without lifecycle hooks or hidden behaviour.

TableGateway wraps a single database table with methods for select, insert, update, and delete operations. It provides a natural place to encapsulate table-specific logic while keeping your code organized and testable.

Unlike heavy ORM solutions, TableGateway doesn't impose conventions or generate queries behind your back. You maintain full control over the SQL while benefiting from a consistent interface. Extend the base class to add domain-specific finder methods for your application.

## RowGateway Pattern

Work with individual database rows as objects with automatic change tracking and persistence.

RowGateway represents a single row from a database table as an object. Access columns as properties, modify them, and call `save()` to persist changes. The gateway tracks which columns have changed and generates efficient UPDATE statements.

New rows can be created, populated, and inserted with the same interface. RowGateway handles primary key management automatically, including support for auto-increment columns and returning the last inserted ID.

## Parameter Binding

Secure prepared statements and consistent handling of parameters across all SQL operations.

All user-supplied values are bound as parameters, never interpolated into SQL strings. This eliminates SQL injection vulnerabilities by design. The library handles type detection and appropriate PDO parameter types automatically.

Named and positional parameters are both supported. When building queries programmatically, parameters are collected and ordered correctly regardless of how the query was constructed. You can also pass raw expressions when you need literal SQL values.

## Pluggable Result Sets

Return arrays, ArrayObjects, custom models, or hydrating results with your own ResultSet type.

Control how query results are returned. The default ResultSet returns associative arrays, but you can configure it to return ArrayObject instances, hydrate custom entity classes, or implement your own result handling logic.

ResultSets are iterable and countable, supporting `foreach` loops and `count()` calls. They can buffer results in memory or stream them directly from the database connection for memory-efficient processing of large datasets.

## Query Profiling

Lightweight instrumentation for capturing executed SQL, bind parameters, and timing information.

Attach a profiler to capture every query executed through the adapter. Each profile includes the SQL string, bound parameters, execution time, and stack trace. This is invaluable for debugging and performance optimization.

The profiler is designed for minimal overhead. Enable it during development or selectively in production to diagnose issues. Export profiles to various formats or integrate with your application's logging infrastructure.

## Metadata Inspection

Retrieve table structures, column definitions, constraints, and indexes programmatically.

Query your database schema at runtime. List tables, examine column types and constraints, discover primary and foreign keys, and inspect indexes. This enables dynamic application behaviour based on database structure.

Metadata is accessed through a consistent interface regardless of the underlying database platform. Results are cached by default to avoid repeated schema queries, with manual cache invalidation when needed.

## DDL Abstraction

Create, alter, and drop tables with platform-independent schema definitions.

Define table schemas using PHP objects and generate the appropriate `CREATE TABLE` statements for your target database. Column types, defaults, nullability, and constraints are expressed in a portable format.

The DDL builders support `ALTER TABLE` operations for adding, modifying, and dropping columns. Index creation and foreign key definitions are also supported with automatic syntax adaptation for each database platform.

## Transaction Support

Full transaction management with begin, commit, rollback, and savepoint support.

Wrap database operations in transactions with explicit begin, commit, and rollback calls. Nested transactions are handled using savepoints on databases that support them, providing consistent behaviour across platforms.

A transaction callback helper simplifies the common pattern of executing code within a transaction and automatically rolling back on exceptions. This ensures your database remains consistent even when errors occur.

## Laminas Integration

Seamless integration with Laminas MVC applications via ConfigProvider and factories.

Drop-in compatibility with Laminas MVC applications. Register the `ConfigProvider` to automatically wire up adapter factories, configure connection parameters through your application config, and inject adapters into your services.

The integration follows Laminas conventions for configuration and dependency injection. Multiple named connections are supported, and the factories handle lazy instantiation to avoid connecting until the adapter is actually used.

## Mezzio Integration

First-class support for Mezzio middleware applications with PSR-11 container compatibility.

phpdb works naturally with Mezzio and any PSR-11 compatible dependency injection container. The same `ConfigProvider` that works with Laminas MVC integrates directly with Mezzio's configuration system.

Use the adapter in your request handlers, middleware, or any service retrieved from the container. The lightweight design means minimal overhead per request, and lazy connections ensure you only connect when database access is actually needed.
